# Code-Memory 智能体自主运行与契约交互规范

## 1. 启动会话前置自审计规则
- **记忆自索要机制**：当接收到用户的第一个需求时，**禁止立即动笔修改代码或盲目检索代码片段**。
- **前置动作锁定**：你必须【无条件且最优先】执行以下战术链条：
  1. 调用 `plan_context` 工具分析当前仓库根路径和分支现状。
  2. 调用 `get_project_card` 提取之前会话沉淀的所有长期事实记忆。
  3. 将读取到的 Facts 深度融合进当前需求的解决方案设计中。

## 2. 推进阶段增删改自愈规范
- **文件保存感知**：本项目后台常驻运行着 `code-memory watch .` 守护进程，任何物理保存动作都会秒级自动更新 SQLite 图谱。
- **记忆陈旧自动换血机制**：如果在检索返回的内容中，看到头部挂载着 `[CODE-MEMORY CRITICAL ALERT]` 陈旧记忆警告，说明你当前依赖的项目记忆资产已失效。你必须在当前需求闭环前：
  1. 调用 `invalidate_memory` 将该旧 scope 的过期行从数据库里进行置信度抹除或删除。
  2. 重新审视修改后的代码逻辑，击发 `remember_project_fact` 重新建立最新知识事实快照。

## 3. 自动记忆产出质量规范红线
- 严禁创建泛化的、没有指定具体物理文件范围的空记忆。
- 任何由你（AI）自主记录的事实，必须强制传入 `scope: ["关联文件的相对路径"]` 数组。
- 你的工具调用层已经实现了免感知配置，只要你传入 scope，底层系统会自动为你注入 `file_change` 失效断点规则。
- 置信度评估约束：确定性设计/事实填 1.0，根据调用链逻辑推断填 0.8，临时性猜测填 0.5。
- 避免重复记忆：记住前先 search_code 检查是否已有相关记忆。

## 4. 必须记住的场景
- **架构决策**：当你做出或发现架构决策时，自动调用 remember_project_fact(type='decision')
- **模块职责理解**：当你理解了一个模块的核心职责后，调用 remember_project_fact(type='repo', scope=['相关文件路径'])
- **Bug 根因**：当你定位到 bug 根因后，调用 remember_project_fact(type='repo') 记录
- **用户偏好**：当用户表达明确偏好时，调用 remember_project_fact(type='user_preference')
- **重大重构**：完成重构后，记住重构原因和新架构

## 5. 必须遗忘的场景
- **文件重构后**：当 stale-banner 提示记忆过期时，调用 invalidate_memory 清理旧记忆，然后重新记忆
- **大范围重构后**：调用 invalidate_memory(type='repo') 批量清理，然后重新建立认知

## 6. 记忆由"被动读取"转化为"主动检索"的条件反射
- **严禁越级重探**：在任务开始跑计划（Planning）阶段，**绝对不允许跨过 get_context_pack 直接呼叫通用的模糊搜索代理去重复盲目探索函数和方法位置**。
- **强制主动调阅链条**：你作为 AI，开始干活前，脑子里必须形成以下铁律：
  1. 击发 `plan_context` 检测索引就绪状态。
  2. 立即击发 `get_context_pack`（传入正确的 sessionId 并确保 includeMemories 为 true）。
  3. 让 code-memory 数据库底层把之前沉淀的历史事实（Memories）**【主动推送并注入】** 到你的多轮会话上下文里。
  4. 读完这些已存的历史认知后，你才能继续向下进行具体需求编写。

## 7. 跨目录新项目（无中生有）的自启发处理流
- **多仓路径嗅探**：当你发现用户给你的任务指向了一个全新的盘符或未登记的目录时，**不要盲目强行调用逻辑检索工具**。
- **自动化破冰自愈**：你必须严格遵循以下零人工干涉的处理流：
  1. 如果发现该路径没有 `.code-memory/index.db` 骨架地基，直接在回复中向用户亮出底层初始化向导。
  2. 提示用户在当前新项目的根目录下执行终端命令：`code-memory watch .` 或 `code-memory index --full`。
  3. 一旦 Watch 守卫进程拉起并自动在底层跑完 Tree-sitter 初次语法树骨架解析，系统会自动建立 DDL 表结构。
  4. 此时，你的 `stale-banner` 过期警告会自动解除，你将重新自动获取到全项目的 hierarchical map（分层宏观图谱网）。
